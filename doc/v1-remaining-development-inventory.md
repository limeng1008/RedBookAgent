# RedBookAgent V1 未完成开发盘点

盘点日期：2026-04-21

当前结论：V1 主闭环已经具备基础可跑能力，不再是从零开发。现在剩余工作主要集中在产品化体验、批量录入、前后端一致性、AI 可观测性、自动化测试和试运营数据。

## 1. 已基本完成

### 1.1 基础模块和菜单

- 已有账号管理、赛道管理、人设管理、产品卖点、提示词模板、敏感词、热点池、选题分析、笔记草稿、发布计划、数据回收、复盘报告等基础 CRUD。
- 已有 Excel 导入导出入口。
- 已有小红书运营相关菜单、工作台、发布日历、复盘看板页面。
- 权限 SQL 已包含多数按钮权限，包括导入、导出、热点分析、草稿审核、风险检查、版本记录、发布计划状态动作、数据完整性检查、复盘回流等。

### 1.2 主闭环接口

主链路接口已经存在：

- 热点分析：`POST /redbook/hotspot/analyze`
- 分析生成草稿：`POST /redbook/hotspotAnalysis/generateDraft`
- 草稿审核通过/退回：`POST /redbook/noteDraft/approve`、`POST /redbook/noteDraft/reject`
- 草稿风险检查：`POST /redbook/noteDraft/riskCheck`
- 草稿版本列表/恢复：`GET /redbook/noteDraft/versions`、`POST /redbook/noteDraft/restoreVersion`
- 草稿加入排期：`POST /redbook/noteDraft/createPublishPlan`
- 发布计划标记已发布/延期/取消/恢复/补录链接：`POST /redbook/publishPlan/markPublished`、`delay`、`cancel`、`restorePending`、`updateNoteUrl`
- 发布计划生成回收记录：`POST /redbook/publishPlan/createMetric`
- 数据回收完整性检查：`GET /redbook/noteMetric/completeness`
- 复盘报告生成/范围生成/回流热点池：`POST /redbook/reviewReport/generate`、`generateScoped`、`createHotspots`
- 工作台和复盘看板：`GET /redbook/workbench/overview`、`reviewDashboard`

### 1.3 状态和关系字段

- 后端已经有状态常量和部分状态流转校验。
- 前端列表、表单、详情里已经使用状态下拉、状态标签。
- 前端大多数关系字段已经改为选择器并显示名称，例如账号、赛道、热点、分析、草稿、发布计划。

### 1.4 AI 接入

- 已有统一 AI 接入层，支持 local / Dify / FastGPT。
- 已有远程 AI 失败后的本地降级逻辑。
- 已有 timeout、retry、错误类型分类、原始结果记录和基础输出结构校验。

## 2. 还没完全开发的地方

### F03 热点池产品化

当前状态：部分完成。

已完成：

- 热点详情、AI 分析按钮、Excel 导入预校验、标题/原文链接重复检查。
- 前端已有批量标废弃入口，但实现方式是逐条调用编辑接口。

待补齐：

- 后端缺少专门的批量标废弃接口，当前前端批量操作不是原子操作。
- 热点评分筛选还不完整，只有普通字段搜索和列表展示。
- 热点池还缺少更运营化的筛选组合，例如赛道、状态、评分区间、采集时间、互动量组合筛选。

### F04 AI 选题分析页

当前状态：部分完成。

已完成：

- 分析详情已有字段展示。
- 已有生成草稿动作。
- 状态字典中已有 `analyzed`、`adopted`、`discarded`。

待补齐：

- 缺少明确的采纳/废弃动作按钮和后端专用接口。
- AI 失败后重试入口不明显，目前主要依赖重新触发分析或生成。
- 评分维度展示还偏字段罗列，缺少更直观的维度卡片或分数解释。

### F06 草稿版本记录

当前状态：大部分完成。

已完成：

- 已建版本实体和接口。
- 新增、编辑、审核、恢复都会留下快照。
- 前端详情抽屉已有版本列表和恢复按钮。

待补齐：

- 版本对比视图未实现，目前只能看单个版本摘要和恢复。
- 恢复前的差异确认还不够清晰。

### F07 内容风险检查

当前状态：部分完成。

已完成：

- 后端已有独立风险检查接口。
- 草稿详情可展示风险检查结果。
- 发布日历抽屉能显示草稿风险检查结果。

待补齐：

- 草稿列表/详情里缺少醒目的“立即风险检查”操作入口。
- 敏感词命中明细、替换建议、风险等级还没有产品化展示。
- 风险检查结果目前更像文本字段，后续应结构化为命中词、位置、建议替换、风险级别。

### F08 发布计划增强

当前状态：部分完成。

已完成：

- 发布日历已能查看排期。
- 日历抽屉支持复制文案、标记已发布、生成回收记录、延期一天、取消排期、恢复待发布、编辑计划。
- 后端已有延期、取消、恢复、补录链接接口。

待补齐：

- 通用发布计划列表页还没有延期、取消、恢复、补录链接这些动作按钮，主要集中在发布日历抽屉。
- 发布提醒未实现。
- 补录链接缺少专门的弹窗体验，当前更依赖编辑表单或接口。

### F09 数据回收增强

当前状态：部分完成，是下一阶段最关键缺口。

已完成：

- 后端能计算互动率、收藏率、评论率。
- 后端已有 2h / 24h / 72h / 7d 完整性检查。
- 数据补齐后会刷新发布计划状态为 `data_collected`。

待补齐：

- 前端还没有完整性检查入口和节点提示展示。
- 还没有批量录入表格，当前主要是通用 CRUD 表单和 Excel 导入。
- 数据回收 Excel 导入缺少逐行预校验错误提示，失败时错误信息比较粗。
- 缺少按发布计划批量录入 2h / 24h / 72h / 7d 的运营工作台式页面。

### F10 复盘看板增强

- 已有报告生成弹窗。
- 已有回流热点池动作。

待补齐：

- 看板 API 仍保留 mock fallback，生产化时应改为真实失败提示。

## 3. 生产化前需要清理的技术债

### 3.1 前端 mock fallback

以下位置还保留 mock fallback：

- `jeecgboot-vue3/src/views/redbook/publish-calendar/publish-calendar.api.ts`
- `jeecgboot-vue3/src/views/redbook/review-dashboard/review-dashboard.api.ts`
- `jeecgboot-vue3/src/views/redbook/crud/redbook.shared.ts`

开发联调阶段可以兜底，但生产化前应改为真实错误提示，避免后端异常时页面显示假数据。

### 3.2 数据回收页不够运营化

通用 CRUD 能用，但数据回收实际工作需要批量录入、节点补齐提醒、缺失项提示、按发布计划聚合展示。这里是最影响试运营效率的缺口。

### 3.3 状态动作不够统一

部分后端接口已存在，但前端入口分散：

- 发布计划延期/取消/恢复主要在日历抽屉，列表页没有。
- 风险检查接口存在，但草稿页操作入口不明显。
- 选题分析采纳/废弃缺少专门动作。

### 3.4 AI 日志不可检索

当前 AI 调用结果可写入业务字段，但缺少独立调用日志。后续排查“为什么这次 AI 生成不好”会比较困难。

## 4. 建议下一步开发顺序

### P0：先补运营闭环硬缺口

1. 数据回收批量录入和完整性提示页面。
2. 草稿页风险检查按钮和结构化风险结果展示。
3. 选题分析采纳/废弃动作。
4. 发布计划列表页补齐延期、取消、恢复、补录链接动作。

### P1：补生产化能力

1. 去掉前端 mock fallback，改为真实错误提示。
2. AI 调用日志表和模板调试入口。
3. 数据回收导入逐行预校验。
4. 草稿版本对比视图。

### P2：补验收和试运营资料

1. 核心接口测试。
2. 前端关键流程自测脚本。
3. 样例数据初始化脚本。
4. 运营人员操作手册和最终部署文档。
