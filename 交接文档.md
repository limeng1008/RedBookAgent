## 4. 数据库同步任务

下一位接手后，先同步数据库。

建议步骤：

```bash
cd /Volumes/aimodel/RedBoolAgent
mysqldump -uroot -proot jeecg-boot > /tmp/jeecg-boot-backup-$(date +%Y%m%d%H%M%S).sql
mysql -uroot -proot < jeecg-boot/db/redbook-agent-schema.sql
```

同步后检查：

```bash
mysql -uroot -proot -e "USE \`jeecg-boot\`; SHOW TABLES LIKE 'rb_note_metric';"
mysql -uroot -proot -e "USE \`jeecg-boot\`; SELECT perms FROM sys_permission WHERE perms LIKE 'redbook:noteMetric:%' ORDER BY sort_no;"
```

注意：

- 本轮新增的 `batchSave` 接口没有新增表、字段，也没有强依赖新按钮权限；它复用 `redbook:noteMetric:edit`
- 仍然要执行 `redbook-agent-schema.sql`，因为之前已有菜单、字典、权限、状态流转相关 SQL 变更，需要保证本地数据库和代码一致
- 如果现有表结构早于当前 schema，`CREATE TABLE IF NOT EXISTS` 不会自动补旧表缺失字段，需要对照 `redbook-agent-schema.sql` 手动补 ALTER

## 5. 后端重启任务

数据库同步后，重启 8080 后端，让新增接口生效。

当前 8080 仍被旧 Java 进程占用：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
```

可以先停止旧进程，再启动新 jar：

```bash
kill 48180
cd /Volumes/aimodel/RedBoolAgent/jeecg-boot
screen -dmS redbook-backend-8080 zsh -lc 'JAVA_HOME=$(/usr/libexec/java_home -v 18) java -jar jeecg-module-system/jeecg-system-start/target/jeecg-system-start-3.9.1.jar --spring.profiles.active=dev --server.port=8080 > /tmp/redbook-backend-8080.log 2>&1'
tail -f /tmp/redbook-backend-8080.log
```

如果 PID 变化，以 `lsof` 查到的实际 PID 为准。
